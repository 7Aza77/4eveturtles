# GoEvent API

GoEvent is a RESTful API service built with Go (Golang) designed for event management and user registrations. The application uses modern patterns, Gin web framework, PostgreSQL for durable data storage, Redis for rate limiting, and JWT for secure authentication.

## Tech Stack
- **Language**: Go
- **Framework**: [Gin](https://gin-gonic.com/)
- **Database**: PostgreSQL (via `sqlx`)
- **Caching / Rate Limiting**: Redis
- **Security**: JWT (`golang-jwt/jwt`)
- **Documentation**: Swagger (`swaggo/gin-swagger`)

## Project Structure
The project follows standard Go layout patterns:
- `cmd/app`: Application entry point (`main.go`)
- `internal/`: Private business logic (handlers, usecases, repositories)
- `pkg/`: Reusable public library code (e.g., Auth/JWT manager)
- `docs/`: Swagger documentation generated by Swaggo

## Getting Started

### Prerequisites
- Docker and Docker Compose
- Or locally: Go installed, alongside PostgreSQL and Redis server instances.

### Running with Docker Compose (Recommended)
The easiest way to start the application along with its PostgreSQL and Redis dependencies is via Docker Compose:

1. Open your terminal in the target directory.
2. Run the following command:
   ```bash
   docker-compose up --build
   ```
3. The API will be exposed on `http://localhost:8080`.

### Local Development Variables
If you are running the service manually (using `go run cmd/app/main.go`), ensure the following environment variables are set (they have local defaults):
- `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`
- `REDIS_HOST`, `REDIS_PORT`
- `JWT_SECRET` (For signing tokens)

## API Documentation
Swagger representation of the API provides a user-friendly UI to explore and test the endpoints. After starting the server, navigate to your `/swagger/index.html` route (if configured in `handler`) to view the interactive documentation.

### Core Features
- **Authentication**: JWT-based user login and registration (`ApiKeyAuth` required for protected routes).
- **Events**: Endpoints for creating, discovering, and administrating events.
- **Event Registration**: Functionality handling user sign-ups for corresponding events.
